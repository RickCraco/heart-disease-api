# Heart Disease Prediction API

A machine learning API that predicts the likelihood of heart disease from patient clinical data. The project demonstrates an end-to-end workflow: data ingestion, model training with hyperparameter tuning, model persistence, and serving predictions through a REST API with input validation and automated tests.

> **Note:** This is an educational portfolio project. It is not intended for clinical or diagnostic use.

## Overview

The application trains an XGBoost classifier on a heart disease dataset and exposes a single prediction endpoint via FastAPI. Preprocessing (scaling and one-hot encoding) is bundled inside a scikit-learn pipeline, so the same transformations are applied consistently during training and inference.

## Features

- **End-to-end ML pipeline** — download data, preprocess features, train, evaluate, and save the model
- **Hyperparameter tuning** — `GridSearchCV` with 5-fold cross-validation
- **REST API** — `POST /predict` endpoint with automatic request/response validation via Pydantic
- **Structured logging** — separate log files for training and API inference
- **Automated tests** — API validation tests and utility function tests with pytest

## Getting Started

### Prerequisites

- Python 3.10 or higher
- `pip`

### Installation

Clone the repository and create a virtual environment from the project root:

```bash
git clone <repository-url>
cd heart-disease-api
python -m venv venv
```

Activate the virtual environment:

```bash
# Windows
venv\Scripts\activate

# macOS / Linux
source venv/bin/activate
```

Install dependencies:

```bash
pip install -r requirements.txt
```

### Train the Model

Training downloads the dataset if it is not already present, runs grid search, evaluates on a hold-out test set, and saves the model to `models/xgboost_pipeline.joblib`.

Run from the project root:

```bash
python -m src.train
```

Training metrics and configuration are written to `logs/training.log`.

### Run the API

Start the server from the project root (the trained model must exist first):

```bash
uvicorn api.main:app --reload
```

The API will be available at `http://127.0.0.1:8000`. Interactive documentation is available at:

- Swagger UI: `http://127.0.0.1:8000/docs`
- ReDoc: `http://127.0.0.1:8000/redoc`

### Run Tests

```bash
pytest
```

For verbose output:

```bash
pytest -v
```

## API Reference

### `POST /predict`

Predict whether a patient is likely to have heart disease.

**Request body**

| Field | Type | Description |
|-------|------|-------------|
| `Age` | int | Age in years (0–120) |
| `Sex` | string | `"M"` or `"F"` |
| `ChestPainType` | string | `"TA"`, `"ATA"`, `"NAP"`, or `"ASY"` |
| `RestingBP` | int | Resting blood pressure in mm Hg (0–200) |
| `Cholesterol` | int | Serum cholesterol in mg/dl (0–600) |
| `FastingBS` | int | Fasting blood sugar > 120 mg/dl: `1` (true) or `0` (false) |
| `RestingECG` | string | `"Normal"`, `"ST"`, or `"LVH"` |
| `MaxHR` | int | Maximum heart rate achieved (60–202) |
| `ExerciseAngina` | string | Exercise-induced angina: `"Y"` or `"N"` |
| `Oldpeak` | float | ST depression induced by exercise (-3.0–7.0) |
| `ST_Slope` | string | `"Up"`, `"Flat"`, or `"Down"` |

**Example request**

```bash
curl -X POST "http://127.0.0.1:8000/predict" \
  -H "Content-Type: application/json" \
  -d '{
    "Age": 58,
    "Sex": "M",
    "ChestPainType": "ASY",
    "RestingBP": 140,
    "Cholesterol": 289,
    "FastingBS": 1,
    "RestingECG": "ST",
    "MaxHR": 110,
    "ExerciseAngina": "Y",
    "Oldpeak": 2.0,
    "ST_Slope": "Flat"
  }'
```

**Example response**

```json
{
  "prediction": 1,
  "predicted_proba": 0.87
}
```

| Field | Type | Description |
|-------|------|-------------|
| `prediction` | int | Predicted class: `0` (no heart disease) or `1` (heart disease) |
| `predicted_proba` | float | Estimated probability of heart disease (0.0–1.0) |

Invalid input returns HTTP `422` with validation details.

## Model Pipeline

1. **Preprocessing** — numerical features are standardised; categorical features are one-hot encoded (`drop="first"`)
2. **Classifier** — XGBoost, tuned via grid search over `n_estimators`, `max_depth`, `learning_rate`, and `gamma`
3. **Evaluation** — 70/30 stratified train/test split; metrics include accuracy (cross-validation), classification report, and ROC AUC on the test set
4. **Persistence** — the full fitted `GridSearchCV` object is serialised with joblib

## Limitations

- Trained on a single public dataset; performance may not generalise to other populations
- No authentication or rate limiting on the API
- Model and logs are stored locally; no cloud deployment included
- Not validated for production or healthcare environments
